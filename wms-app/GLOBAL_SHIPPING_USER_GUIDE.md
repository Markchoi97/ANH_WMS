## 1. 드롭시핑 (Drop Shipping)

### 📌 용도
해외 플랫폼(淘宝, Shopify, Shopee 등)에서 들어온 주문을 국내 창고에 입고하고 관리합니다.

### 🔧 기능
- Excel/CSV 파일 일괄 업로드
- 물류사 연동 (한진, CJ, 顺丰 등)
- 주문 정보 편집
- 패킹 리스트 출력

### 📖 사용 방법

#### STEP 1: 템플릿 다운로드 및 작성
1. **"템플릿 다운로드"** 버튼 클릭
2. Excel 파일에 주문 정보 입력:
   ```
   주문번호 | 고객사 | SKU | 상품명 | 수량 | 목적국가 | 배송방법
   수취인 | 전화번호 | 주소 | 도시 | 우편번호 | 물류사 | 비고
   ```
3. 다국어 지원:
   - 한글: 주문번호, 고객사, 수취인...
   - 중문: 订单号, 客户, 收件人...
   - 영문: Order ID, Customer, Receiver...

#### STEP 2: 파일 업로드
1. **"Excel/CSV 업로드"** 버튼 클릭
2. 작성한 파일 선택
3. 자동으로 데이터 매핑 및 테이블 표시

#### STEP 3: 주문 편집 (필요시)
1. 각 주문 옆의 **연필 아이콘(✏️)** 클릭
2. 편집 가능 항목:
   - **기본 정보**: 주문번호, 고객사, SKU, 수량
   - **물류 정보**: 물류사 선택, 운송장 번호 입력
   - **수취인 정보**: 이름, 전화번호, 주소, 도시, 우편번호
   - **비고**: 특이사항 메모
3. **"저장"** 버튼 클릭

#### STEP 4: 패킹 리스트 출력
1. 출력할 주문 선택 (체크박스)
2. **"패킹 리스트 출력"** 버튼 클릭
3. 인쇄 미리보기 확인 후 출력
4. 패킹 리스트 내용:
   - 바코드 (주문 ID)
   - 주문 정보 (주문번호, 고객사, SKU, 수량)
   - 수취인 정보 (이름, 전화번호, 주소)
   - 물류사 및 운송장 번호

#### STEP 5: 일괄 입고 처리
1. 모든 정보 확인 완료 후
2. **"일괄 입고 처리"** 버튼 클릭
3. 자동으로 재고 차감 및 운송장 번호 부여

### 💡 팁
- **바코드 스캔 모드**: 개별 입고 시 바코드 스캔으로 빠르게 처리
- **물류사 선택**: 수취인 국가에 따라 적절한 물류사 선택 (한국→한진/CJ, 중국→顺丰)
- **대량 주문**: 100건 이상 주문 시 여러 번 나눠서 업로드 권장

### ⚠️ 주의사항
- 수취인 정보는 **반드시 정확하게** 입력 (배송 오류 방지)
- 운송장 번호는 자동 생성되나 수동 입력도 가능
- SKU가 재고에 없으면 입고 처리 실패 (오류 메시지 확인)

## 2. 2차 정렬 (Second Sorting)

### 📌 용도
1차 창고 단위 분류 후, SKU별로 세부 분류하고 수취인별로 패키지를 구성합니다.

### 🔧 기능
- 바코드 스캔으로 자동 매칭
- 중복/누락 자동 검출
- 물류사별 파렛트 자동 구성
- 실시간 진행률 추적

### 📖 사용 방법

#### STEP 1: 스캔 모드 시작
1. **"바코드 스캔"** 버튼 클릭
2. 스캔 모드 활성화 (초록색으로 변경)

#### STEP 2: 주문 스캔
1. 주문번호 또는 SKU 바코드 스캔
2. 자동으로 주문 매칭 및 상태 업데이트:
   - ⚪ 대기 → 🟡 진행 → 🟢 완료
3. 스캔 수량이 주문 수량과 일치하면 자동 완료

#### STEP 3: 파렛트 확인
1. 우측 **"파렛트 구성"** 패널에서 확인
2. 물류사별로 자동 그룹핑됨:
   - 한진택배: PLT-hanjin
   - CJ대한통운: PLT-cj
   - 顺丰速运: PLT-shunfeng
3. 각 파렛트에 포함된 주문 수 및 상품 개수 표시

#### STEP 4: 파렛트 마감
1. 한 파렛트의 주문이 모두 완료되면
2. **"파렛트 마감"** 버튼 클릭
3. 마감된 파렛트는 출고 준비 완료 상태로 전환

### 💡 팁
- **중복 스캔 방지**: 같은 상품을 여러 번 스캔하면 🔴 오류로 표시
- **진행률 확인**: 상단 진행률 바로 전체 작업 진행도 파악
- **물류사 필터**: 특정 물류사만 보고 싶을 때 필터 사용

### ⚠️ 주의사항
- 스캔 전 주문 수량 확인 (수량만큼 스캔 필요)
- 파렛트 마감 후에는 수정 불가
- 이상 건이 있으면 빨간색으로 표시되니 먼저 처리

## 3. 패키지 검증 (Package Check)

### 📌 용도
포장 단위(2B, 2S 등)를 확인하고, 무게를 측정하며, 운송 라벨을 인쇄합니다.

### 🔧 기능
- 포장 타입 선택 (2B, 2S, BOX, ENVELOPE, CUSTOM)
- 무게 자동 기록
- 운송 라벨 인쇄 (바코드 포함)
- 무게 오차 자동 감지 (±5%)

### 📖 사용 방법

#### STEP 1: 패키지 스캔
1. **"바코드 스캔"** 버튼 클릭
2. 패키지 ID, 주문번호, 또는 운송장 번호 스캔
3. 현재 작업 중인 패키지로 표시됨

#### STEP 2: 무게 측정
1. 전자저울에 패키지 올리기
2. **실제 무게** 입력 (kg 단위)
3. **"무게 기록"** 버튼 클릭
4. 예상 무게와 비교:
   - 오차 ±5% 이내: ✅ 정상
   - 오차 ±5% 초과: ⚠️ 경고 (재확인 필요)

#### STEP 3: 라벨 인쇄
1. **"라벨 인쇄"** 버튼 클릭
2. 인쇄 미리보기 확인
3. 라벨 내용:
   - 물류사 이름
   - 운송장 번호 (바코드 포함)
   - 수취인 정보
   - 상품 정보 (SKU, 수량)
   - 포장 타입 및 무게
4. A4 용지 또는 라벨 전용지에 인쇄

#### STEP 4: 검증 완료
1. 라벨을 패키지에 부착
2. **"검증 완료"** 버튼 클릭
3. 출고 준비 완료 상태로 전환

### 💡 팁
- **포장 타입**: 
  - 2B (30x25x20cm): 중형 박스
  - 2S (25x20x15cm): 소형 박스
  - ENVELOPE: 서류, 의류 등
- **무게 오차**: 5% 이상 차이 나면 재측정 권장
- **라벨 부착 위치**: 박스 상단 중앙에 부착 (바코드가 잘 보이게)

### ⚠️ 주의사항
- 무게 측정 전 저울 영점 조정 필수
- 라벨 인쇄 후 반드시 바코드 스캔 테스트
- 무게 오차가 크면 내용물 누락 가능성 확인

## 4. 무게 측정 (Weight Check)

### 📌 용도
출고 전 최종 무게를 측정하고, 물류사별 운임을 자동으로 계산합니다.

### 🔧 기능
- 바코드 스캔으로 패키지 식별
- 자동 무게측정 모드 (저울 연동)
- 운임 자동 산출 (물류사별 요금표)
- 중량 오차 감지 (±5%)
- 물류사별 통계

### 📖 사용 방법

#### STEP 1: 자동측정 모드 설정 (선택)
1. **"자동측정"** 버튼 클릭 (전자저울 연동 시)
2. 자동측정 ON 상태 확인
3. 스캔만 하면 자동으로 무게 측정 (현재 시뮬레이션)

#### STEP 2: 패키지 스캔
1. **"바코드 스캔"** 버튼 클릭
2. 주문번호, 운송장 번호, 또는 패키지 ID 스캔
3. 현재 측정 중인 패키지 표시

#### STEP 3: 무게 입력
1. 전자저울에서 무게 확인
2. **실제 무게** 입력 (소수점 2자리까지)
3. 자동으로 **예상 운임** 미리보기 표시
4. **"기록"** 버튼 클릭

#### STEP 4: 운임 확인
1. 측정 완료 후 자동으로 운임 계산
2. 물류사별 요금:
   - **한진택배**: ₩3,000 + ₩500/kg
   - **CJ대한통운**: ₩3,500 + ₩550/kg
   - **顺丰速运**: ₩8,000 + ₩2,000/kg (국제특송)
   - **EMS**: ₩5,000 + ₩1,500/kg
3. 우측 통계 패널에서 물류사별 총 운임 확인

### 💡 팁
- **자동측정 모드**: 전자저울 API 연동 시 스캔만으로 측정 완료
- **최소 중량**: 각 물류사마다 최소 요금 적용 중량 있음 (보통 0.5kg)
- **운임 통계**: 실시간으로 총 운임 확인하여 물류 비용 관리

### ⚠️ 주의사항
- 중량 오차 ±5% 초과 시 내용물 재확인 필수
- 운임은 추정치이며, 실제 요금은 물류사 정산 시 확정
- 무게는 반올림하여 kg 단위로 계산 (0.6kg = 1kg)

## 5. 교환/반품 (Exchange/Return)

### 📌 용도
고객의 반품/교환 요청을 접수하고, 검수하여, 적절한 방법으로 처리합니다.

### 🔧 기능
- 반품/교환 요청 조회
- 7가지 반품 사유 관리
- 상품 검수 및 상태 판정
- 사진 첨부 (증빙 자료)
- 4가지 처리 방법 선택
- 승인/거절 프로세스

### 📖 사용 방법

#### STEP 1: 요청 조회
**방법 1: 바코드 스캔**
1. **"바코드 스캔"** 버튼 클릭
2. 주문번호 또는 운송장 번호 스캔

**방법 2: 검색**
1. 검색창에 주문번호, 운송장, SKU, 또는 고객명 입력
2. 해당 요청 클릭

#### STEP 2: 검수 시작
1. 요청 상세 화면에서 **"검수 시작"** 버튼 클릭
2. 반품 상품 수령 확인
3. 반품 사유 확인:
   - 🔧 불량/하자
   - 📦 오배송
   - 📏 사이즈 불일치
   - 💭 단순 변심
   - 💔 파손/손상
   - ❓ 미배송/분실
   - 📝 기타

#### STEP 3: 상태 판정
상품 상태를 정확히 판정:
- **✅ 정상**: 재판매 가능, 포장 완전
- **🔧 불량**: 제조 결함, 기능 이상
- **💔 파손**: 운송 중 파손, 외관 손상
- **❓ 분실**: 상품 누락, 내용물 불일치

#### STEP 4: 사진 첨부
1. **"클릭하여 사진 업로드"** 영역 클릭
2. 다음 사진 촬영:
   - 불량/파손 부위 클로즈업
   - 전체 상품 사진
   - 포장 상태 사진
3. 여러 장 업로드 가능

#### STEP 5: 처리 방법 선택
상태에 따라 적절한 처리 방법 선택:

| 상태 | 권장 처리 방법 |
|------|---------------|
| ✅ 정상 | 📦 재고 복귀 / 🔄 교환 발송 |
| 🔧 불량 | 🗑️ 폐기 처리 / 💰 환불 처리 |
| 💔 파손 | 🗑️ 폐기 처리 / 💰 환불 처리 |
| ❓ 분실 | 💰 환불 처리 |

**처리 방법 설명:**
- **📦 재고 복귀**: 정상 상품을 재고에 다시 추가 (자동 반영)
- **🗑️ 폐기 처리**: 불량/파손 상품 폐기 (재고 제거)
- **🔄 교환 발송**: 동일/다른 상품으로 교환 배송
- **💰 환불 처리**: 고객에게 금액 환불

#### STEP 6: 승인/거절
1. 모든 정보 확인 후
2. **승인** 또는 **거절** 버튼 클릭
3. 승인 사유:
   - 반품 사유가 타당함
   - 상품 상태 확인 완료
   - 정책 범위 내
4. 거절 사유:
   - 반품 기한 초과
   - 상품 훼손 (고객 과실)
   - 사용 흔적 있음

#### STEP 7: 처리 완료
1. 승인된 요청은 선택한 방법대로 처리
2. **"처리 완료"** 버튼 클릭
3. 자동으로:
   - 재고 복귀 시 → 재고 수량 증가
   - 폐기 처리 시 → 재고에서 제거
   - 환불 처리 시 → 환불 금액 기록

### 💡 팁
- **빠른 검수**: 바코드 스캔으로 즉시 조회 가능
- **사진 필수**: 불량/파손 건은 반드시 사진 첨부 (분쟁 방지)
- **처리 통계**: 상단 통계 카드로 전체 현황 파악

### ⚠️ 주의사항
- 검수는 반드시 **꼼꼼하게** (재판매 품질 보장)
- 단순 변심 반품은 **상품 상태 확인** 필수
- 사진 첨부 누락 시 추후 분쟁 발생 가능
- 재고 복귀 전 상품 재포장 확인

```
2. 📋 상품 준비 (개발 예정)
   ↓ 환적지 선택 → 라벨 부착 → 출고 대기 목록
   
3. 📊 출고 계획(파도 관리) (개발 예정)
   ↓ SKU 자동분류 → Batch 생성 → 분류상태 모니터링
   
4. 📦 2차 정렬
   ↓ 바코드 스캔 → 중복 검출 → 파렛트 구성 → 마감
   
5. 🔍 검증/검사 (개발 예정)
   ↓ 이미지 캡처 → 이상품 업로드 → 재고 반영
   
6. ✅ 패키지 검증
   ↓ 무게 측정 → 라벨 인쇄 → 송장 부착 → 검증 완료
   
7. ⚖️ 무게 측정
   ↓ 바코드 스캔 → 무게 입력 → 운임 산출 → 기록
   
8. 🔄 교환/반품
   ↓ 검수 → 상태 판정 → 사진 첨부 → 처리 선택 → 완료
   
9. ⚠️ 이상 처리 (개발 예정)
   ↓ 자동탐지 → 보고서 생성 → 알림 전송
   
10. ⏰ 마감 시간 (개발 예정)
    ↓ 마감시간 설정 → 미처리건 리마인드 → 운송사별 안내
```

모든 페이지에서 **"바코드 스캔"** 버튼으로 빠른 작업 가능:

- 운송장 번호

- 패키지 ID

- 🚚 한진택배
- 📦 CJ대한통운
- 🎁 롯데택배
- ⚡ 顺丰速运 (SF Express)
- ✈️ EMS 우편

### 자동화 기능
1. **재고 자동 차감**: 입고 처리 시
2. **파렛트 자동 구성**: 물류사별 자동 그룹핑
3. **운임 자동 산출**: 무게 기반 실시간 계산
4. **중복/누락 자동 감지**: 스캔 시 실시간 체크
5. **재고 자동 반영**: 반품 재고 복귀 시

## 자주 묻는 질문 (FAQ)

### Q1. Excel 파일 업로드 시 오류가 발생해요
**A:** 다음을 확인하세요:
- 템플릿 형식을 사용했는지 확인
- 열 이름이 정확한지 확인 (한글/중문/영문 모두 가능)
- 필수 항목이 비어있지 않은지 확인
- 수량은 숫자로만 입력

### Q2. 바코드 스캔이 안 돼요
**A:** 
- 바코드 스캐너가 USB로 연결되어 있는지 확인
- 스캔 모드가 활성화(초록색)되어 있는지 확인
- 바코드가 깨끗하고 손상되지 않았는지 확인
- 수동으로 주문번호를 검색창에 입력해도 됩니다

### Q3. 물류사를 잘못 선택했어요
**A:**
- 드롭시핑 페이지에서 해당 주문의 **✏️ 편집** 버튼 클릭
- 물류사 다시 선택 후 저장
- 아직 출고 전이라면 언제든 수정 가능

### Q4. 무게 오차가 계속 나요
**A:**
- 전자저울 영점 조정 필수
- 포장 재료 무게 포함 여부 확인
- 예상 무게가 부정확할 수 있음 (실제 무게가 정확)
- ±5% 이내면 정상 범위

### Q5. 파렛트를 잘못 마감했어요
**A:**
- 현재는 마감 취소 기능이 없습니다
- 관리자에게 문의하여 수동으로 처리
- 차후 업데이트에서 취소 기능 추가 예정

### Q6. 반품 상품을 재고에 넣지 않고 싶어요
**A:**
- 검수 시 상태를 **🔧 불량** 또는 **💔 파손**으로 판정
- 처리 방법을 **🗑️ 폐기 처리** 선택
- 재고에 반영되지 않음

### Q7. 운송장 번호를 수동으로 입력하고 싶어요
**A:**
- 드롭시핑 페이지에서 주문 편집
- 운송장 번호 필드에 직접 입력
- 자동 생성된 번호를 덮어쓸 수 있음

### Q8. 중국 고객사를 위한 중문 지원이 되나요?
**A:**
- 네! Excel 템플릿에서 중문 열 이름 지원
- 订单号(주문번호), 客户(고객사), 收件人(수취인) 등
- 시스템에서 자동으로 인식

### Q9. 패킹 리스트를 PDF로 저장하고 싶어요
**A:**
- 패킹 리스트 출력 시 인쇄 미리보기 화면
- "대상"을 "PDF로 저장" 선택
- 원하는 위치에 저장

### Q10. 한 번에 몇 개까지 주문을 처리할 수 있나요?
**A:**
- 제한은 없지만, 100~200건씩 나눠서 처리 권장
- 대량 처리 시 브라우저 성능 고려
- 서버 부하 분산을 위해 적절히 나눠서 업로드
