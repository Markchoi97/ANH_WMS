## 시스템 개요

### 🎯 ANH WMS란?

ANH WMS는 창고 운영의 모든 프로세스를 디지털화한 통합 관리 시스템입니다.

### ✨ 주요 기능

- **실시간 재고 추적** - 제품 입출고 현황을 실시간으로 확인
- **작업 자동화** - 입고/출고/포장 작업을 자동으로 배정
- **국제 배송 연동** - 중국/한국 주소 자동 파싱 및 배송사 자동 배정
- **데이터 시각화** - 한눈에 보는 운영 현황 대시보드

### 🌐 접속 방법

1. 웹 브라우저 실행 (Chrome, Edge, Safari 권장)
2. 회사에서 제공한 URL 접속
3. 로그인 정보 입력

## 대시보드

> **위치:** 홈 화면 (로그인 직후)

### 📊 한눈에 보는 운영 현황

대시보드는 창고 운영의 핵심 지표를 실시간으로 보여줍니다.

#### 1️⃣ 통계 카드 (상단)

**총 제품 수**
- 시스템에 등록된 전체 제품 종류
- 클릭 시 재고 관리 페이지로 이동

**총 재고 수량**
- 현재 보유 중인 전체 재고 개수
- 모든 제품의 수량 합계

**재고 부족 품목**
- 최소 재고량 이하인 제품 개수
- 🔴 빨간색으로 경고 표시
- 즉시 발주 필요

**금일 입/출고**
- 오늘 처리된 입고/출고 건수
- 형식: `입고 건수 / 출고 건수`

#### 2️⃣ 최근 입고 내역

최근 5건의 입고 정보를 표시합니다.

**표시 정보:**
- 제품명
- 공급업체
- 수량 (초록색 +표시)
- 입고 일시

**활용 방법:**
- 입고 처리 확인
- 공급업체별 입고 패턴 파악

#### 3️⃣ 최근 출고 내역

최근 5건의 출고 정보를 표시합니다.

**표시 정보:**
- 제품명
- 고객사
- 수량 (파란색 -표시)
- 출고 일시

**활용 방법:**
- 출고 처리 확인
- 고객사별 주문 패턴 파악

#### 4️⃣ 재고 부족 경고

최소 재고량 이하 제품 목록을 표시합니다.

| 제품명 | SKU | 현재 재고 | 최소 재고 | 위치 |
|--------|-----|----------|----------|------|
| 키보드 | KEY-001 | 8개 | 15개 | A-2-06 |

**조치 사항:**
1. 🔴 빨간색 수량 확인
2. 공급업체 발주 진행
3. 입고 예정 확인

## My Tasks

> **위치:** 좌측 메뉴 > My Tasks (NEW 뱃지)

### ✅ 나의 작업 관리

내가 처리해야 할 작업들을 관리하는 개인 작업 페이지입니다.

#### 📊 작업 통계

상단에 작업 현황이 표시됩니다:
```
내 작업 목록
오늘 해야 할 8개의 작업이 있습니다

⚪ 예정 3건  |  🟡 진행 3건  |  🟢 완료 2건  |  🟣 보류 0건
```

#### 📝 작업 카드

각 작업은 카드 형태로 표시됩니다:

```
┌────────────────────────────────────────┐
│ 🔴 높음    🟡 진행 중                  │
│                                        │
│ 키보드 입고 확인                       │
│ 기계식 키보드 30개 입고 확인 필요      │
│                                        │
│ 📦 키보드 × 30개                      │
│ 📍 A-2-06                             │
│ ⏰ 16:00까지                          │
│                                        │
│ [시작] [스캔] [완료] [보류]           │
└────────────────────────────────────────┘
```

#### 🎯 우선순위

| 색상 | 우선순위 | 의미 |
|------|---------|------|
| 🔴 | 높음 | 긴급 처리 필요 |
| 🟡 | 중간 | 일반 작업 |
| ⚪ | 낮음 | 여유 있게 처리 |

#### 🔘 작업 버튼

##### 1️⃣ **시작 버튼**
- 작업을 시작할 때 클릭
- 상태가 "⚪ 예정" → "🟡 진행 중"으로 변경
- 시작 시간 자동 기록

##### 2️⃣ **스캔 버튼**
- QR코드/바코드 스캔
- 제품 정보 자동 확인
- 위치 검증

**스캔 프로세스:**
```
1. [스캔] 버튼 클릭
2. 카메라 활성화
3. QR/바코드 스캔
4. 자동 매칭 확인
   ✅ 일치: 다음 단계
   ❌ 불일치: 경고 메시지
```

##### 3️⃣ **완료 버튼**
- 작업 완료 시 클릭
- 확인 메시지 표시
- 상태가 "🟢 완료"로 변경
- 완료 시간 자동 기록

##### 4️⃣ **보류 버튼**
- 작업을 일시 중단할 때
- 보류 사유 입력 필수
- 사진 첨부 가능 (불량, 재고 부족 등)

**보류 사유 예시:**
- 재고 부족
- 불량품 발견
- 위치 불일치
- 기타 (직접 입력)

## 주문업로드&배송연동

> **위치:** 좌측 메뉴 > 주문업로드&배송연동 (NEW 뱃지)

### 📦 자동화된 주문 처리 시스템

엑셀 파일을 업로드하면 자동으로 국가를 인식하고 배송사를 배정합니다.

---

### 📤 주문 업로드 탭

#### 📋 엑셀 템플릿

필수 컬럼 (중국어/한국어 모두 지원):

| 중국어 헤더 | 한국어 헤더 | 설명 | 필수 |
|-----------|-----------|------|------|
| 订单号 | 주문번호 | 고유 주문 번호 | ✅ |
| 收件人姓名 | 수취인 | 받는 사람 이름 | ✅ |
| 收件人电话 | 전화번호 | 연락처 | ✅ |
| 收件地址 | 주소 | 배송 주소 | ✅ |
| 收件人邮编 | 우편번호 | 우편번호 | ⭕ |
| 商品名称 | 상품명 | 제품명 | ⭕ |
| 备注 | 비고 | 특이사항 | ⭕ |

#### 📝 엑셀 작성 예시

```
订单号         | 收件人姓名 | 收件人电话        | 收件地址                        | 收件人邮编
A2025-000001  | 张三       | +86-138-0000-1111 | 上海市 浦东新区 花木路100弄5号  | 200120
A2025-000002  | 김철수     | 010-2222-3333     | 서울시 강남구 테헤란로 123      | 06000
```

#### 🚀 업로드 프로세스

1. **파일 준비**
   - Excel (.xlsx, .xls) 또는 CSV 파일
   - 첫 번째 시트만 읽음
   - 헤더는 첫 번째 행

2. **업로드**
   - 드래그 & 드롭 또는 클릭하여 파일 선택
   - 자동 업로드 시작

3. **자동 처리**
   ```
   📄 파일 읽기
      ↓
   🔍 주소 분석 (중문/한글 감지)
      ↓
   🌍 국가 인식 (CN/KR/기타)
      ↓
   🚚 배송사 자동 배정
      ↓
   💾 데이터 저장
      ↓
   📊 결과 표시
   ```

4. **결과 확인**
   - ✅ 성공: 초록색 카드
   - ❌ 실패: 빨간색 카드 + 실패 사유

#### ⚠️ 업로드 주의사항

- 중복 주문번호는 자동 거부
- 필수 컬럼 누락 시 실패
- 한 번에 최대 1,000건 권장

### 🚚 배송사 자동배정 탭

#### 🤖 자동 배정 규칙

```
중국 주소 감지 (중문자 포함)
   ↓
📦 ANH 물류 배정

한국 주소 감지 (한글 포함)
   ↓
🚛 CJ 대한통운 배정

기타 국가
   ↓
✈️ 국제 배송 배정
```

#### 📊 통계 카드

4개의 통계 카드가 표시됩니다:

```
┌──────────┬──────────┬──────────┬──────────┐
│ 전체     │ CJ 대한통운 │ ANH 물류  │ 국제 배송 │
│ 150건    │ 80건      │ 60건     │ 10건     │
└──────────┴──────────┴──────────┴──────────┘
```

클릭하면 해당 배송사의 주문만 필터링됩니다.

| 주문번호 | 수취인 | 국가 | 배송사 | 상품명 | 등록일시 |

| A2025-001 | 张三 | 🇨🇳 CN | ANH 물류 | 노트북 | 2025-11-02 14:30 |
| A2025-002 | 김철수 | 🇰🇷 KR | CJ 대한통운 | 마우스 | 2025-11-02 14:35 |

#### 📊 API 로그 상세

주문을 클릭하면 우측에 상세 로그가 표시됩니다:

```
┌─────────────────────────────────┐
│ API 처리 로그                   │
├─────────────────────────────────┤
│ ✅ CJ - REQUEST                │
│ HTTP Status: 200                │
│ 2025-11-02 14:30:15            │
│ [상세 보기 ▼]                  │
├─────────────────────────────────┤
│ ❌ CJ - RESPONSE               │
│ HTTP Status: 400                │
│ 오류: 주소 형식 불일치          │

│ [상세 보기 ▼]                  │
└─────────────────────────────────┘
```

**상세 보기 클릭 시:**
- 전체 API 요청/응답 JSON 표시
- 오류 원인 파악 가능

#### 🔧 문제 해결 가이드

| 오류 유형 | 원인 | 해결방법 |
|----------|------|---------|
| 주소 형식 오류 | 주소가 불완전 | 주소 수정 후 재업로드 |
| 중복 주문 | 이미 등록됨 | 기존 주문 확인 |
| 배송사 연동 실패 | API 오류 | 잠시 후 재시도 |
| 타임아웃 | 네트워크 지연 | 자동 재시도 대기 |

**입력 항목:**

| SKU | 고유 코드 | LAP-001 | ✅ |

| 단위 | 수량 단위 | 개 | ✅ |
| 최소 재고 | 경고 기준 | 10 | ✅ |
| 단가 | 가격 | 1,200,000원 | ✅ |
| 위치 | 창고 위치 | A-1-01 | ⭕ |

**재고 부족 표시:**
- 🔴 빨간색: 최소 재고 이하
- ⚠️ 경고 아이콘

#### 🗑️ 제품 삭제

## ❓ FAQ

### Q1. 로그인이 안돼요
**A:** 다음을 확인해주세요:
- 이메일 주소가 정확한가요?
- 비밀번호에 오타가 없나요?
- 계정이 활성화되어 있나요?

### Q2. 재고가 마이너스로 표시돼요
**A:** 출고가 입고보다 많을 때 발생합니다.
- 입고 내역 확인
- 누락된 입고 등록
- 관리자에게 문의

### Q3. 엑셀 업로드가 실패해요
**A:** 다음을 확인해주세요:
- 필수 컬럼이 모두 있나요?
- 헤더가 첫 번째 행에 있나요?
- 파일 형식이 .xlsx 또는 .csv인가요?

### Q4. QR코드 스캔이 안돼요
**A:** 
- 카메라 권한을 허용했나요?
- 조명이 충분한가요?
- QR코드가 손상되지 않았나요?

### Q5. 데이터가 실시간으로 안 보여요
**A:** 
- 페이지 새로고침 (F5)
- 브라우저 캐시 삭제
- 인터넷 연결 확인
